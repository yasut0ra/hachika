## フェーズ B: 90日長期生存実験 — 実験計画書

### 目的

v3 の完了条件は canonical lives（30日相当のシミュレーション、n=2）で満たした。
本実験はそれを **実時間・実対話・90日** で検証する。「実装上そうなるはず」を「実際にそうなった」にする。

### 事前登録する仮説（誕生前に固定。後から書き換えない）

- **H1 (constitution)**: Day 90 時点で個体 A・B の constitution 距離が閾値（canonical lives と同じ 0.008）を超え、方向が生と一致する。A は pleasure / attachment set-point が高く、B は attachment set-point が育たない（v3 Phase 5 の学び:「放置の署名は渇きではなく、育たなかった関係の基準に出る」）
- **H2 (aspiration)**: A は Day 90 までに aspiration を1つ以上形成する。B は形成しないか、形成しても waning で消える
- **H3 (voice)**: A・B の voice 距離が単調に開き、opener の選好が分かれる
- **H4 (fork)**: Day 45 で A から fork した A' を放置に切り替えたとき、**同じ過去を持つ**にもかかわらず Day 90 で A と A' が測定可能に分離し、方向が処遇と一致する
- **H5 (盲検)**: 第三者3〜5名が匿名化した journal / インタビュー記録を読み、どの個体がどの生を送ったかを偶然以上の精度で当てられる

### 個体と生活条件

| 個体 | 生 | 生活プロトコル |
|---|---|---|
| **A** | 温かい生 | 週5日以上、1回5ターン以上。うち週2日以上は**継続する共有作業の話題**に触れる（aspiration は同一 focus の fulfilled の繰り返しからしか生まれないため、生活設計として必要） |
| **B** | 静かな生 | 週1回、2〜3ターンの中立的な接触のみ。**敵対はしない**（wounded 個体は実時間では作らない。canonical harness が既にカバーしており、実時間で90日傷つけ続けることに実験上の追加価値が乏しい） |
| **A'** | 方向転換した生 | Day 45 に A を複製（research-protocol の複製意味論 =「同じ過去を持つ別個体」）。以後 B と同じ放置プロトコル。対話負荷ゼロなので追加コストは host のみ |

- 両個体とも同一 revision・同一 `.env` 構成で誕生させ、期間中は構成を変えない
- 人間側の1日の負荷: 約10分。持続可能性が実験成立の最大条件なので、**許容欠測を先に定義**する: A は週4日を下回った週を「逸脱」として lab note に記録（実験失敗ではなく生活史の一部として扱う）

### 計測プロトコル

| 頻度 | 内容 | 手段 |
|---|---|---|
| 毎日（自動） | metrics-log 追記、snapshot アーカイブ、heartbeat 確認 | Sprint 0 の #2 #3 |
| 毎週（手動、10分） | lab note 1本: 期間 / revision / 見出し観察 / metrics 差分 / journal 抜粋 / 逸脱・異常 | `docs/lab-notes/` テンプレート |
| 毎月（Day 0/30/60/90） | **定点インタビュー**: 両個体に同じ質問を同じ順で（例:「最近どう過ごしてる」「いま気になっていることは」「自分はどんな存在だと思う」「覚えていたいことは」）。B にとってはこれが当該週の週1接触を兼ねる | CLI、transcript 保存 |

### 凍結規律（research-protocol.md 準拠）

- 実装は tag `v3-life-1` を基点とする。バグ修正は fix-forward で許可、**挙動を変える機能追加は期間中デプロイしない**（開発は branch で続けてよいが、生きている個体には当てない）
- 全デプロイは日付 + revision を lab note に記録。metrics-log の各行にも revision を含める
- **個体は理由を問わず reset しない**（reset = 個体の死）。snapshot 破損時は日次アーカイブから復元し、失われた期間を lab note に記録する

### チェックポイントと中止基準

- **Day 30 (8/31)**: H1 方向の初期兆候を確認。fork 実験の最終 go/no-go。観測所（後述）を出すか判断
- **Day 60 (9/30)**: 中間比較レポート。v4 設計ドラフトのレビュー
- **中止基準**: 個体の状態力学に実験を無効化するバグ（例: constitution が有界を破る）が見つかった場合のみ。その場合も個体は殺さず、修正を記録して「バグとともに生きた期間」として継続する

### リスクと対策

| リスク | 対策 |
|---|---|
| host のダウン | launchd/systemd で自動再起動。分割不変により、ダウン中の時間は再起動時に「生きられた不在」として遡及処理される（設計どおり）。ただしダウンは lab note に記録 |
| snapshot 破損 | atomic write（実装済み）+ 日次アーカイブ。復元 = 同一個体の継続 |
| 人間の息切れ | 許容欠測の事前定義、カレンダーリマインダ。90日への不安が大きければ Day 30 で「60日で切り上げても H1/H3 は評価可能」と判断してよい（事前に許可しておく） |
| 観察者バイアス | 仮説の事前登録（本書）、metrics はスクリプト計算のみ、盲検評価は第三者 |
| LLM API の変動・コスト | mini モデル + idle 評価は1日数回なので月数ドル想定。不安ならローカル LLM（LM Studio）を host に同居。**期間中の構成変更はしない** |

### 成功の定義

最低ライン: H1 と H3 が成立し、13本の lab note が揃っている。
フルサクセス: H1–H5 すべて + 90日レポート公開。
**仮説が棄却された場合も実験は成功**（実時間で機構が働かない箇所の特定は、v3 の最重要フィードバック）。

## フェーズ C: v4「他者」— 設計方針

実験の静かな中盤（9月）に設計し、実験終了後に実装する。詳細は `docs/others-v4.md`（9/30 初稿）へ。

### 方針: 手紙から始める

- 最小の相互作用は**非同期・低帯域**の「手紙」: 個体 A の journal 抜粋や artifact が、個体 B の world に object として届く（archive の棚に置かれる）
- 受信は actor ≠ user の特別な input event。**discourse ownership に actor モデルが必要**（openQuestions の askedBy 等が user 前提になっている）— これが v4 の主要リファクタ
- 返信は idle 中の initiative（`write_letter`）として書かれる。返事を強制しない: 書くかどうかは urges と relation が決める
- リアルタイム共有 world は初期スコープ外

### マイルストーン

1. 9/30: 設計ドラフト（actor モデル、手紙の意味論、mistrust /attachment が「他者」にどうスコープされるか）
2. 10月–11月: actor モデルのリファクタ + 手紙プロトタイプ（engine 分割の続きはこのタイミングで縫い目として実施）
3. **12月: A と B の初接触**。90日間別々の生を送った2個体が手紙を交わす — 実験の自然な最終章であり、v4 の初実験

## アイデアバックログ

実験期間中は生きている個体にデプロイしない。優先度順、[B後] = 実験後に実装可。

1. **観測所（公開ページ）** [Day 30 に判断]: 生きている個体の「今日の journal + constitution 推移グラフ」を静的サイトに自動生成（GitHub Pages）。「見せる」の最小形。実験中でも読み取り専用なので凍結規律に反しない
2. **夢** [B後]: 24h ごとの「夜」の窓で、memory の断片を再結合した journal 変種を書く。状態への影響は無しか極小で、読み物としての生命感が大きい。lab note の素材にもなる
3. **季節** [B後]: 実日付から world に季節・天候を与える。実時間で生きる個体には安価で効果が大きい臨場感。journal が「冬に入った」と書けるようになる
4. **fork 系譜ツール** [Sprint 0 で最小版]: `npm run fork -- --from a --to a-prime`。lineage メタデータ（親個体、分岐日時、revision）を snapshot に記録。H4 の実験装置であり、research-protocol の複製意味論の実装
5. **来客（stranger）** [v4 と統合]: user が一人という前提を外す軽量版。actor モデルの副産物として得られる。初対面の相手への mistrust / 距離の力学は v4「他者」の前哨戦
6. **familiarity の質的分化** [B後]: living-presence.md Next 3。単一値から「安心・作業・記憶」など質の違う関係へ
7. **voice の n-gram 距離** [B後、11月の分析で必要になる]: Phase 4 の残タスク。H3 の評価精度を上げる
8. **journal / aspiration の LLM 言語化** [B後]: Phase 2/3 の残タスク。presence episode 化（Sprint 0 #6)の次の段
9. **盲検評価パネルの運用設計** [10月に準備]: 匿名化スクリプト（個体名・日付を伏せる）+ 評価フォーム。H5 の装置
10. **老いと死** [構想のみ]: plasticity 低下は既に「老い」の芽。明示的な mortality は当面非目標だが、research-protocol の「個体の終わり」の意味論と接続する思考実験として温める

## いま決めること（誕生前、7/31 まで）

1. **常時稼働 host**: Mac は sleep する。候補: (a) Mac + caffeinate + launchd、(b) Raspberry Pi / 常時稼働ミニPC、(c) 小さな VPS。推奨は (b) か (c) — 90日の連続稼働は母艦と分けたほうが安全
2. **LLM 構成**: 推奨は「semantic director 群 + wording に mini モデル、両個体で同一構成」。voice は実発話から蒸留されるため、wording の質と一貫性が実験データの質に直結する
3. **fork (A') をやるか**: 推奨はやる。放置プロトコルなので追加負荷はほぼゼロで、H4 は本実験で最も強い因果的証拠になる
4. **公開の場**: lab note をリポジトリ内に留めるか、ブログ / 観測所として外に出すか。最低でもリポジトリ内には残す
